## 1. 化祿轉忌：尋找「祿的歸宿」與「定數」

梁式名言：「祿是因，忌是果。」祿轉忌的過程，就是在看「老天給你的好處，最後變成了什麼？」

### 核心象義邏輯：
* **承接與轉化：** A 宮化祿入 B 宮，代表 B 宮「得到」了這個好處。但 B 宮不能獨吞，它必須透過其天干轉忌到 C 宮，這代表 B 宮把這個好處「用」在哪裡。
* **吉凶判斷標準（關鍵）：**
    * **吉（成格）：** 若 C 宮（終點）是**同類宮位**（如命三方、田宅），代表這個好處最後「入庫」或「歸我所有」。
    * **凶（虛有其表）：** 若 C 宮轉忌後**沖**了命三方或田宅，這叫「祿入忌出」，代表機會來了卻守不住，空歡喜一場。

### 程式邏輯中的象義分類：

| **祿轉忌入同類宮** | 圓滿、得財、延續。例如：命祿入財，財轉忌入田（錢賺了拿回家存）。 |
| **祿轉忌入異類宮** | 轉移、消耗。例如：命祿入官，官轉忌入交友（事業的好處最後都給了別人）。 |

## 2. 化忌轉忌：追蹤「苦的蔓延」與「坑洞」

化忌代表壓力、執著、糾紛、虧欠。忌轉忌則是「一波未平一波又起」，或是「為了填補 A 坑，結果挖了 B 坑」。

### 核心象義邏輯：
* **執著的延伸：** A 忌入 B，代表 A 宮對 B 宮有虧欠或壓力。B 轉忌入 C，代表這個壓力讓 B 宮也扛不住了，繼續把問題拋向 C。
* **損失的擴大：** 忌轉忌通常帶有「不得已」的色彩。如果說祿轉忌是在找「成果」，忌轉忌就是在找「導火線」與「災情範圍」。

| :--- | :--- |
| **忌轉忌（同類相纏）** | 糾纏不清、深陷其中。例如：夫妻忌入命，命轉忌入夫妻（兩人互欠，分不開也過不好）。 |

## 2. `checkAggregation()` 程式碼實作

```javascript
/**
 * 梁氏飛星：多祿會合搜尋引擎
 * @param {Array} palaces - 包含 12 宮位天干與坐星的陣列
 * @returns {Object} 每個宮位的能量統計與來源
 */
function checkAggregation(palaces) {
    // 初始化 12 宮的能量統計
    let energyMap = Array.from({ length: 12 }, () => ({
        count: 0,
        sources: [], // 哪些宮位化祿過來
        finalPaths: [] // 轉忌後的終點
    }));

    // 第一階段：掃描全盤「化祿」
    for (let i = 0; i < 12; i++) {
        const stem = palaces[i].stem;
        const luStar = liangFourTransform[stem]["祿"];
        const targetIdx = findStarPalace(luStar); // 假設您已有此函數找星曜宮位

        if (targetIdx !== -1) {
            // 記錄第一層化祿
            energyMap[targetIdx].count += 1;
            energyMap[targetIdx].sources.push({ from: i, type: '直接化祿' });

            // 第二階段：執行「轉忌」 (梁氏核心)
            const bStem = palaces[targetIdx].stem;
            const jiStar = liangFourTransform[bStem]["忌"];
            const finalIdx = findStarPalace(jiStar);

            if (finalIdx !== -1) {
                // 能量轉移至終點
                energyMap[finalIdx].count += 1;
                energyMap[finalIdx].sources.push({ from: i, via: targetIdx, type: '祿轉忌匯聚' });
            }
        }
    }

    // 過濾出有「多祿會合」的宮位 (能量 >= 2)
    const aggregations = energyMap
        .map((data, idx) => ({ palaceIdx: idx, ...data }))
        .filter(item => item.count >= 2);

    return {
        allEnergy: energyMap,
        topAggregations: aggregations.sort((a, b) => b.count - a.count)
    };
}
```
